# 聊天室示例

这是一个使用 flare-core 实现的完整聊天室示例，展示了以下核心功能：

## 功能特性

### 服务端
- 支持 WebSocket 和 QUIC 双协议
- 集成带认证功能的连接管理器
- 实现基本的聊天室功能：广播用户消息
- 当有用户加入或退出时，向所有在线用户发送通知
- 支持用户认证流程

### 客户端
- 使用 ProtocolRacer 实现协议竞速功能
- 支持用户认证流程
- 能够发送聊天消息和接收来自其他用户的消息
- 能够接收并显示用户加入/退出的通知

## 快速开始

### 1. 生成 TLS 证书

确保你已经生成了 TLS 证书，如果没有，请运行：

```bash
./scripts/generate_certs.sh
```

### 2. 启动服务端

```bash
cargo run --example chat_room_server
```

服务端将监听以下地址：
- WebSocket: 127.0.0.1:9005
- QUIC: 127.0.0.1:9006

### 3. 启动客户端

在新的终端窗口中运行：

```bash
cargo run --example chat_room_client <用户名> <认证令牌>
```

示例：
```bash
cargo run --example chat_room_client user1 token_user1
cargo run --example chat_room_client user2 token_user2
```

## 使用说明

### 服务端命令

- `/users` - 显示在线用户列表
- `/help` - 显示帮助信息
- `/quit` - 退出服务端

### 客户端命令

- `/users` - 请求显示在线用户列表
- `/quit` - 退出聊天室

## 认证机制

聊天室使用简单的令牌认证机制：

1. 新连接需要在 30 秒内完成认证
2. 客户端需要提供有效的用户名和令牌
3. 未认证的连接只能发送认证消息
4. 认证超时的连接将被自动断开

## 技术实现

### 消息格式

聊天消息使用 JSON 格式序列化：

```json
{
  "sender": "用户名",
  "content": "消息内容",
  "timestamp": 1234567890
}
```

### 协议支持

- **WebSocket**: 适用于 Web 应用和浏览器客户端
- **QUIC**: 提供更低的延迟和更好的性能

### 连接管理

使用 `AuthenticatedConnectionManager` 管理连接的认证状态：

1. 新连接首先进入待认证状态
2. 客户端发送认证消息
3. 服务端验证令牌
4. 认证成功后连接可以正常通信
5. 认证失败或超时的连接将被断开
