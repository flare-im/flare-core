# 交互式聊天演示使用指南

## 📋 概述

这是一个完整的交互式长连接聊天演示，展示了 flare-core 在实时通信场景中的应用。

## ✨ 功能特点

- 🔗 **长时间连接**：WebSocket 长连接保持
- 💬 **实时交互**：用户输入即时发送，服务端实时响应
- 🔄 **双向通信**：客户端和服务端可以互相发送消息
- 🚪 **优雅退出**：输入 `exit` 安全退出
- 📊 **实时统计**：显示在线用户数和消息总数
- 🎨 **彩色输出**：清晰区分不同类型的消息

## 🚀 运行演示

```bash
cargo run --example interactive_chat
```

## 💡 使用方法

### 1. 启动程序

运行命令后，程序会自动：
- 启动 WebSocket 服务端（监听 `127.0.0.1:9002`）
- 启动客户端并连接到服务端
- 显示欢迎消息

### 2. 发送消息

在提示符 `你 >` 后输入消息，按回车发送：

```
你 > 你好，服务器！
```

### 3. 接收响应

服务端会回复带有服务端标记的消息：

```
📨 [服务端] [服务端回复 #1] 收到您的消息: "你好，服务器！" | 在线: 1 | 总消息: 1

你 > 
```

### 4. 退出程序

输入 `exit` 并回车：

```
你 > exit

👋 正在退出聊天室...
🔔 [系统消息] 再见！感谢使用 Flare 聊天室 👋
```

## 🔧 技术细节

### 架构设计

```
┌─────────────┐                    ┌─────────────┐
│   客户端    │ ←── WebSocket ──→  │   服务端    │
│             │                    │             │
│ - 用户输入  │                    │ - 消息处理  │
│ - 消息发送  │                    │ - 回复生成  │
│ - 响应接收  │                    │ - 状态管理  │
└─────────────┘                    └─────────────┘
```

### 消息格式

使用 JSON 序列化的 `ChatMessage` 结构：

```rust
{
    "id": 1,
    "sender": "Client",
    "content": "你好",
    "timestamp": 1697500000,
    "msg_type": "user"
}
```

消息类型：
- `user` - 用户消息
- `server` - 服务端响应
- `system` - 系统消息

### 关键特性

1. **异步 I/O**
   - 使用 Tokio 异步运行时
   - 非阻塞的用户输入读取
   - 并发处理发送和接收

2. **消息编解码**
   - 使用 flare-core 的 `MessageParser`
   - JSON 序列化
   - 二进制传输（WebSocket Binary frame）

3. **优雅关闭**
   - 检测 `exit` 命令
   - 发送退出确认
   - 清理资源

4. **状态管理**
   - 服务端维护活跃连接数
   - 消息计数器
   - 使用 `Arc<RwLock>` 共享状态

## 📝 注意事项

1. **输入缓冲**：由于使用异步读取标准输入，某些终端可能需要按两次回车
2. **退出延迟**：输入 `exit` 后会等待 500ms 以接收服务端的退出确认
3. **单客户端**：当前演示为单客户端模式，可以扩展为多客户端

## 🐛 故障排除

### 问题：无法连接到服务端

**解决**：
- 检查端口 9002 是否被占用
- 确保防火墙允许本地连接
- 查看错误日志

### 问题：输入没有响应

**解决**：
- 确认已按回车键
- 检查终端是否支持异步输入
- 尝试重启程序

### 问题：消息乱码

**解决**：
- 确保终端使用 UTF-8 编码
- 检查系统区域设置
